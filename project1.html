Case Study

HATCH

A Cloud Storage Solution for Architects and Their Clients

Research | Brand | Prototype

PROBLEM

Create a cloud storage solution that meets the needs of a particular audience and leverages the network effect.

SOLUTION

Hatch, a cloud storage application for architects and their clients that allows users to upload, share, organize, and create. Hatch also features pinboard style social media to leverage the network effect.

COMPETITIVE ANALYSIS & BRAND STRATEGY

SWOT Table

A SWOT analysis was performed on Google Drive, Dropbox, and Pinterest

  • Google Drive does storage and collaboration well but lacks in organization.
  • Dropbox shares visual files well but is lacking in a centralized community and risks users switching to anotehr application with more options.
  • Pinterest is great for organization, storage of interests, and networking but lacks in document sharing and storage.

Strategy

Create a cloud storage application geared toward a profession, that shares and stores files with a built in organizational system, featuring a pinboard style social media feed.

The field I chose to gear the applications toward was architecture. There is a need in that profession to share and organize large files, which can also serve as a place to connect with clients.

USER TESTING & ITERATIONS

Initial Designs

Through user testing and preference testing, initial desiging was transformed from an oversimplified version to a version with great flow and detail.

Updates:

  • Call to action buttons changed from grey to green
  • Ditched Montserrat Subrayada as a font option because personas-users found the underlining confusing
  • Took the logo off of the dark background
  • Upload process was streamlined to one step
  • Font sizes were scaled down
  • Main pages received titles
  • Redesigned pages to address spacing issues
Full Mockups

Future Considerations

Hatch in its current state is a minimally viable product. If the product was developed I would implement the following:

  • Version control
  • Document security
  • Direct linking to relevent design software

CONCLUSION

Since Hatch was my first full scale project I grew by leaps and bounds, but there were three key revelations that struck me during this project.

First, minimalism is harder to appropriately apply than I had initially thought, and thoughful design is key.

Next I realized there is a danger that when you are overly familiar with a product, expecially if you created it, to assume user knowledge and create confusion.

Additionally I leared to seek feeback constantly and aggressively. Testing each process, each flow, each design decision, will help keep the needs of the user forefront.
